Preserve ref.func declaration sources in RemoveUnusedModuleElements

RemoveUnusedModuleElements kept function bodies but could still prune the
only non-function declaration source that made a retained ref.func legal.
That produced validation failures where the offending function body showed
no diff because the invalidation came from removing a declaration-only
global, table initializer, or element segment elsewhere in the module.

Fix the pass by collecting ref.func targets that remain in kept function
bodies and preserving only the globals, table initializers, and element
segments that still declare those functions. Re-run the analysis worklists
and type-dependency marking after preserving those declaration carriers so
index remapping stays consistent.

Add regressions covering both declaration-only globals and declarative elem
segments so root_all_functions mode keeps the required declaration sources
for retained ref.func users.
