# Out-of-Scope Knowledge Base

Use `.out-of-scope/` to store durable records of rejected enhancement ideas.

## Purpose
1. preserve institutional memory
2. deduplicate repeat feature requests

## Rules
- one file per concept, not per issue
- use short kebab-case filenames
- explain *why* the request is out of scope
- prefer durable reasoning over temporary circumstances

## Write to `.out-of-scope/` only when
- the issue is an enhancement
- the maintainer decides it is `wontfix`

Then either append to the existing concept file or create a new one, comment on the issue, and close it with `wontfix`.
