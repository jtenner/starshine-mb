# Write a Skill

When authoring a new skill:
1. define the trigger and description clearly
2. write the workflow in direct imperative steps
3. keep the scope narrow and reusable
4. add supporting reference docs only when they materially help
5. prefer durable guidance over project-specific trivia
6. test the skill by checking whether a fresh agent would know when and how to use it
