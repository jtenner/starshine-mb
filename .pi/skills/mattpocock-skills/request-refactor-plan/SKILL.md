# Request Refactor Plan

When the user asks for a refactor plan:
1. Clarify the problem being solved.
2. Identify current pain points and risks.
3. Define target outcomes and non-goals.
4. Break the work into safe, reviewable stages.
5. Call out dependencies, migration concerns, and test strategy.
6. Produce a concise plan the user can approve or turn into an issue.
