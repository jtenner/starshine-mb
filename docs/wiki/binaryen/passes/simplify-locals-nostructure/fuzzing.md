The stable aliases `simplify-locals-nostructure`, `simplify-locals-nostructure-closeout`, `simplify-locals-nostructure-all-profiles`, `slns`, and `slns-closeout` resolve to the same aggregate.

## Profile set

`src/validate/gen_valid.mbt` defines one aggregate plus three leaf profiles:

| Profile | Weight in aggregate | Intended SLNS surface |
| --- | ---: | --- |
| `simplify-locals-nostructure-straight-line` | 3 | first-cycle single-use sink, equivalent-get cleanup, and root dropped-tee cleanup without effectful barriers |
| `simplify-locals-nostructure-tee-control` | 2 | root-level dropped tee with unrelated structured control, plus a no-structure barrier shape that should not synthesize result-producing control |
| `simplify-locals-nostructure-effect-order` | 2 | read-only load movement, overwritten load/dead-load local traffic, global/table barriers, calls/call_ref/call_indirect context from the effect-capable generator envelope |
| `simplify-locals-nostructure-all` | composite | deterministic weighted selection of the three leaves; manifests keep `config_label` as the aggregate and record the sampled `selected_profile` |

The profiles intentionally reuse the existing local-traffic body-generator gate so the manifest also records the established `has_ssa_*` local-shape feature facts. This is generator plumbing reuse, not a claim that SLNS is the SSA no-merge pass.
