# Pass Tracing Playbook

Guidance for adding trace-friendly optimizer and validator output in Starshine.

## Goals
- Keep tracing cheap when disabled.
- Keep output machine-parseable.
- Preserve function-level + phase-level timing and helper counters.
- Keep behavior reproducible across reruns.

## 1) Trace Interface
- Every traced entrypoint should use:
  - `trace? : (String) -> Unit = <pass>_trace_noop`
  - `trace_all_funcs? : Bool = false`
- For no-op wrappers:
  - `trace_func = pass_trace_log_enabled(ordinal, trace_all_funcs)`
  - `func_trace = if trace_func { fn(msg) { trace(msg) } } else { <pass>_trace_noop }`
- Module-level tracing should guard all timing work with a local `trace_enabled` check.

## 2) Log Contract
- Log as `key=value` tokens.
- Required minimum keys:
  - `pass[...]`:start/done/error
  - `func[...]` lifecycle
  - `phase=...` (function-local and helper-level)
  - `changed=true|false`
  - elapsed time (`elapsed_ms=` or `*_us=`)
  - one or two cardinality counters (`locals=`, `top_instrs=`)
- Use one compact `pass[*]:done` summary line.

## 3) Timing and Counters
- Use shared helpers in `src/lib/util.mbt`:
  - `trace_now_ms/us`
  - `trace_elapsed_ms/us_since`
  - `trace_delta_us_to_ms`
- Pass helpers should expose per-function snapshots:
  - `snapshot`, `zero`, `diff`, `emit`
- Collect both:
  - per-function deltas
  - run-level totals in `Map[String, UInt64]` and `Map[String, Int]`

## 4) Hotspots and Guardrails
- Track top-N expensive functions by call-time and size metrics.
- Emit one final sorted hotspot summary per pass.
- Trace suppression:
  - emit the first K detailed failures, then a single `suppressed` summary.
- Optional pathology guard for expensive paths:
  - report `budget_exceeded=true` and chosen fallback/revert action.

## 5) Optimizer / Validator Integration
- Prefix pass logs with a stable pass name:
  - `simplify_locals`, `vacuum`, `merge_blocks`, etc.
- Tracing gates should emit:
  - `pass[start]`, `phase[start/done/error]`, `pass[done/error]`
- Validator tracing should mirror this contract for module-level and function-level phases.

## 6) Validation Rule
- Do not add trace-only tests.
- Trace behavior is validated indirectly through existing functional/integration tests.
- If a change affects trace shape, prefer unit-level contract checks in passing tests.

## 7) Integration Checklist
1. Add traced and non-traced entrypoints.
2. Add optional trace parameters.
3. Add stable function ordinals + per-function gate.
4. Emit start/phase/done/error lines with consistent keys.
5. Add helper snapshots and per-function deltas.
6. Emit function/hotspot summaries plus run totals.
7. Ensure errors preserve totals and helper deltas before returning.
8. Wire tracing in `optimize.mbt` or equivalent dispatcher.
