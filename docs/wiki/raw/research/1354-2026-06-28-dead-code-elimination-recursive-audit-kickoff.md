## Recursive slice contract

Initial recursive continuation threads used three atomic slices per handoff. Per the latest user update, every recursive continuation thread from this point forward should attempt exactly **two** atomic slices before evaluating completion and handing off again. A good atomic slice is one of:

1. source/status classification update for a bounded transform family;
2. red-first focused test(s) for one family or explicit boundary;
3. implementation/fix for that family;
4. GenValid profile leaf/metadata addition plus generator test;
5. focused compare/replay classification for one family;
6. docs/backlog/fuzzing update tied to just-completed behavior.

Do not spend a thread only reading unless blocked. Leave the repo better after each thread.
