# SGO v0.1.0 signoff

## Decision

`simplify-globals-optimizing` is signed off for the current Starshine v0.1.0 supported direct-pass / nested-runtime / late-tail scheduling surface.

This is not a claim that Starshine has implemented every Binaryen `SimplifyGlobals.cpp` rewrite family. Future breadth remains possible, but it is no longer an active v0.1.0 correctness or preset-scheduling blocker without new evidence of a semantic mismatch, wasm validation failure, or a concrete user-facing need.

Whole-command `--optimize` wall time is deliberately outside this signoff. A full public preset artifact compare was attempted after preset widening but proved infeasible in this environment and was stopped by user decision. Future full-preset runtime work belongs under `[WALL]001` unless a measurement attributes the cost specifically to SGO.

## Direct SGO evidence

Direct debug-artifact command:

```sh
bun scripts/self-optimize-compare.ts \
  tests/node/dist/starshine-debug-wasi.wasm \
  --simplify-globals-optimizing \
  --out-dir .tmp/sgo-direct-debug-artifact-nested-pruned \
  --starshine-bin _build/native/release/build/cmd/cmd.exe
```

Result:

- Starshine output was valid.
- Starshine size: `2,860,269` bytes.
- Binaryen size: `2,861,435` bytes.
- Starshine pass-local runtime: `153.143ms`.
- Binaryen pass-local runtime: `107.210ms`.
- Pass-local runtime is inside the 2x floor.
- First canonical diff: `defined=48 abs=69`.
- Accepted classification: representation-only local/default-init drift. Binaryen preserves one explicit `local.set $0 (i32.const 0)`, while Starshine relies on WebAssembly default local zero-initialization.

Nested cleanup trace:

- `.tmp/sgo-nested-pruned-trace.txt`
- total SGO pass time: `139.507ms`
- `detail:sgo:nested-total=86.104ms`
- largest remaining nested wrapper: `vacuum=45.538ms`

Standard direct fuzz:

```sh
bun fuzz compare-pass \
  --count 10000 \
  --seed 0x5eed \
  --pass simplify-globals-optimizing \
  --max-failures 20 \
  --keep-going-after-command-failures \
  --out-dir .tmp/pass-fuzz-sgo-nested-pruned-10k
```

Result: `9975/10000` compared, `9975` normalized matches, `0` mismatches, `0` validation failures, and `25` Binaryen/tool command failures.

The broader SGO page records the many earlier focused 10k lanes for same-init, read-only-to-write, runtime propagation, reference/externref guardrails, typed replacement guardrails, and nested cleanup scheduler characterization.

## Late-tail evidence

The accepted post-SGO neighborhood is:

```text
simplify-globals-optimizing
-> remove-unused-module-elements
-> string-gathering
-> reorder-globals
-> directize
```

Direct late-tail artifact replay at `.tmp/sg-tail-neighborhood-baseline` validated and stayed smaller than Binaryen (`2,856,748` vs `2,861,120` bytes). The first diff `defined=39 abs=60` appears immediately after `simplify-globals-optimizing -> remove-unused-module-elements` and is carried forward unchanged by `string-gathering`, `reorder-globals`, and `directize`.

Same-input RUME isolation was canonical-green on both SGO-side artifact inputs:

- `.tmp/rume-cross-sgo/self-compare-star-sgo-rume`
- `.tmp/rume-cross-sgo/self-compare-bin-sgo-rume`

Standard ordered-neighborhood fuzz:

```sh
bun scripts/pass-fuzz-compare.ts \
  --count 10000 \
  --seed 0x5eed \
  --pass simplify-globals-optimizing \
  --pass remove-unused-module-elements \
  --pass string-gathering \
  --pass reorder-globals \
  --pass directize \
  --max-failures 20 \
  --keep-going-after-command-failures \
  --jobs auto \
  --out-dir .tmp/pass-fuzz-sg-tail-neighborhood-10k \
  --starshine-bin _build/native/release/build/cmd/cmd.exe
```

Result: `9972/10000` compared, `9972` normalized matches, `0` mismatches, `0` validation failures, and `28` Binaryen/tool command failures.

Public `optimize` and `shrink` now append the accepted suffix. Preset scheduling evidence is recorded in [`0572`](./0572-2026-05-19-public-preset-late-tail-scheduling.md).

## v0.1.1 follow-up opportunities

These are no longer active v0.1.0 blockers by themselves and are now tracked as v0.1.1 backlog slices in `agent-todo.md`:

- `[SGO]004` nested cleanup runtime and exact-scheduler experiment: exact Binaryen nested default-function scheduler parity despite measured runtime cost, optional remaining nested `vacuum` wrapper overhead reduction, and cheaper function-filtered adapters for currently module-shaped cleanup passes.
- `[SGO]005` default-local compare normalization: compare-harness or canonicalization work for the accepted explicit-default-local-initialization diff.

Resume any of these only when there is a new semantic mismatch, validation failure, targeted artifact/code-size need, string/GC/refinalization requirement, release-QA need for quieter diffs, or measured SGO-specific wall-time owner.
