# IR2 Test Matrix

## Scope

- Define the shared fixture and golden-coverage layer for IR2 hot bodies, analyses, lowering, analysis caches, and pass traces.
- Keep future slices on one deterministic test harness instead of ad hoc per-module fixtures.
- Keep the living navigation surface in [`wiki/ir2/test-matrix.md`](../../ir2/test-matrix.md) synchronized with this handoff doc when the matrix shape changes.
- Current in-tree evidence is captured in [`wiki/raw/ir2/2026-05-20-ir2-test-matrix-repository-refresh.md`](../ir2/2026-05-20-ir2-test-matrix-repository-refresh.md).

## Shared Helpers

- [`src/ir/test_helpers.mbt`](../../../../src/ir/test_helpers.mbt):
  - `ir_test_build_hot_from_wat(...)` parses WAT, validates the module, lifts a selected defined function, and verifies the hot body.
  - `ir_test_assert_verify_and_lower(...)` verifies a mutated hot body, lowers it into the original module shell, and validates the result.
  - `ir_test_dump_cfg(...)`, `ir_test_dump_dom(...)`, `ir_test_dump_liveness(...)`, and `ir_test_dump_ssa(...)` produce deterministic cross-analysis text dumps.
- [`src/passes/pass_test_helpers.mbt`](../../../../src/passes/pass_test_helpers.mbt):
  run the public pass pipeline from WAT fixtures, lift first-function fixtures through the shared checkpoint, and capture deterministic pass traces.

## Current Golden Fixtures

- CFG / dominance / liveness / SSA:
  a local-phi `if` fixture in [`src/ir/test_helpers_test.mbt`](../../../../src/ir/test_helpers_test.mbt).
- Hot mutation and lower/validate roundtrip:
  a `nop` cleanup fixture plus a Binaryen-style tee/compare branch shape in [`src/ir/test_helpers_test.mbt`](../../../../src/ir/test_helpers_test.mbt).
- Pass traces:
  a single-pass `vacuum` trace in [`src/passes/trace_golden_test.mbt`](../../../../src/passes/trace_golden_test.mbt).

## Correctness Rules

- Golden output must stay deterministic across test runs.
- Shared helpers must use public lift/lower, analysis, cache, mutation, and pass APIs.
- Analysis overlays are revision-keyed caches, not owned IR; tests should prove cache reuse and rebuild where cache behavior is the subject.
- Trace goldens may assert revision numbers only when the mutation path is deterministic.
- White-box tests are for package-internal invariants; durable optimizer behavior should still have public-path or pass-path coverage.
- Add new shared dump helpers only when multiple tests need the same durable cross-analysis output.

## Validation Plan

- Run `moon test src/ir` for IR helper, analysis, cache, lift/lower, builder, query, walk, mutation, or region-helper changes.
- Run `moon test src/passes` for pass helper or trace changes.
- Before commit for behavior or API changes, run:
  - `moon info`
  - `moon fmt`
  - `bun validate readme-api-sync`
  - `moon test`
