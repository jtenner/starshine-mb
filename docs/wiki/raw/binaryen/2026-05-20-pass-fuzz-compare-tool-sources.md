## Durable takeaways

- `bun fuzz compare-pass` is not an ordinary fuzz suite. It is a Binaryen-oracle comparison task that may call `src/fuzz --emit-gen-valid-batch` to prepare local inputs, but its result semantics live in `scripts/lib/pass-fuzz-compare-task.ts`.
- The harness compares **normalized text**, not raw bytes. The flow validates generated input, runs Starshine, validates Starshine output, runs Binaryen `wasm-opt`, canonicalizes both outputs with `wasm-opt --all-features --strip-debug`, prints both with `-S`, and only then compares text.
- Validation failures, generator failures, command failures, and normalized mismatches are distinct statuses in `cases.jsonl` / `result.json`. Do not report all non-match statuses as semantic mismatches.
- Command failures are classified so known tool/parser/oracle failures, such as Binaryen empty-recursion-group errors, can be replayed and discussed separately from Starshine validation failures or normalized output drift.
- `--jobs >1` deliberately requires `--starshine-bin` because otherwise multiple workers would invoke `moon run` concurrently and contend on the shared Moon build lock.
- Failure directories are replay units. Each persisted failure carries copied artifacts, `failure.txt`, best-effort `input.print.wat`, and `failure-metadata.json` with case index, generator, detail, artifact list, relative input, and pass flags.

## Uncertainties and boundaries

- The harness is an oracle-comparison aid, not a proof that every Binaryen-visible representation difference is semantic. A normalized mismatch still needs pass-owner judgment and, when necessary, reduced replay evidence.
- `wasm-smith` is a high-value external generator but can expose Binaryen or Starshine tool gaps. The harness therefore validates generated inputs and keeps generator/tool failures separate from pass mismatches.
- `gen-valid` inputs use Starshine's portable coverage-forced configuration through the batch emitter. That makes them useful for Binaryen comparison, but they should not be confused with the natural `validate-valid` fuzz profile.
