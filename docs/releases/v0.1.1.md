> **Status as of 2026-07-21:** not publication-ready. The July 21 DAE/DAEO results below are retained as a strong validation and performance baseline, but the implementation has been reopened because the authoritative completion matrix still contains `generic-partial` and `mixed` source-family ownership. Release-blocking work is tracked under `[O4Z-DAE]001` in `agent-todo.md` and detailed in `docs/wiki/binaryen/passes/dead-argument-elimination/reopened-parity-plan.md`.

- Establishes a strong Binaryen-v131 DAE and `dae-optimizing` checkpoint for the currently generated and retained-artifact surfaces; it is no longer treated as full source-family closeout.
- Restores exact late preset placement for `heap-store-optimization -> redundant-set-elimination -> vacuum -> dae-optimizing -> inlining-optimizing`.

- Complete generic boundary/exposure and incoming-parameter live-in facts.
- Route every signature, constant, localization, GC, result, control, and type mutation through common epoch-bearing transactions.
- Complete uniform constants, immutable globals, recursive forwarding SCCs, GC parameter/result LUBs, multivalue observation, tail dependencies, bottom repair, recursive-group/type liveness, and metadata policy.

- Port or map all represented Binaryen-v131 DAE source-test families, renew deterministic and four-lane evidence, rerun artifacts/runtime/idempotence/performance, and reconcile all stale completion wording.

## Compatibility notes

- No public MoonBit or Node API is proposed for removal by this reopen.
- Plain DAE remains direct-only; optimizing nested cleanup must not leak into the plain aliases.
- Unknown indirect/reference callers, indirect/reference tail calls, continuations, and unsupported exceptional/type proposal forms may remain conservative, explicitly tested barriers.
- The remaining shared nested local-layout/remap gap and pre-DAEO large O4z wall-time owner stay under neighboring-pass ownership.
- Publication remains human-controlled. This release preparation does not publish npm packages, create tags, or push credentials-sensitive registry changes.
