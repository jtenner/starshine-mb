# Starshine v0.1.1 — 2026-07-21

## Highlights

- Completes the Binaryen-v131-compatible plain DAE and `dae-optimizing` boundary family for the supported Starshine surface.
- Restores exact late preset placement for `heap-store-optimization -> redundant-set-elimination -> vacuum -> dae-optimizing -> inlining-optimizing`.
- Shares Binaryen-v131's touched `precompute-propagate -> default O4z function pipeline` roster with the inlining scheduler.
- Cuts retained DAEO artifact productive/idempotent medians from `47.956s` / `44.490s` to `25.440s` / `21.475s` without changing output bytes.

## Changed

- Replaced numeric function identities, fixed definition lists, caller bands, productive caps, and most module-cardinality admission with exact boundary, call, value-slice, dependency, epoch, and touched-function evidence.
- Removed the `4096` small/broad immutable-global split. Exact materializable call-target discovery now works at every module size.
- Stable inactive-callsite uniform analysis is now module-size independent.
- Retained `1024/4096`, touched-count, function-size, and encoded-size policies are optional phase/profitability budgets only. Mandatory boundary repair and validated cheap cleanup run independently.
- Synchronized `moon.mod` and `node/package.json` at version `0.1.1`.

## Fixed

- Prevented a removed mixed-type parameter read from aliasing an unrelated body local when cached use evidence was stale. The retained plain-DAE artifact now validates and completes instead of emitting an `i64`/`i32` mismatch.
- Added touched mandatory unreachable-tail repair after signature changes so inactive callsite cleanup does not depend on optional nested replay budgets.
- Reused epoch-stable large-uniform call facts until a commit and avoided dependency-graph construction for empty dropped-result sets.
- Restored nested-directory Node package builds after the MoonBit workspace migrated from `moon.mod.json` to `moon.mod`.
- Made the packaged WASI runner route `/dev/stdout` and `/dev/stderr` through its configured file descriptors, and made the wasm CLI trim its program-name argument like the native CLI. The optimized WASI startup smoke now passes on Node v26.
