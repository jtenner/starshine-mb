> This note does not change package versions. The repository remains on `0.1.1`; do not tag or publish without explicit release approval and closure or explicit acceptance of the performance blocker below.

## Current DAE/DAEO closure

- Boundary-changing producers commit through epoch-bearing work outcomes with producer-owned effects, exact touched accounting, validation, rollback, and domain invalidation.
- Plain DAE and DAEO share the Binaryen-v131 boundary engine while preserving variant separation.
- DAEO uses one shared `precompute-propagate`-first nested roster and one effect frontier; private mini-rosters, recursive core replay, duplicate cleanup ownership, and whole-pass retry are removed.
- Cleanup-deleted dependencies remain visible by seeding the frontier from both pre-cleanup and post-cleanup dependency graphs.
- Private zero-reachable-call cleanup is restricted to terminal-unreachable boundaries; ordinary returning functions and represented dropped-self dependencies remain fail closed.
- Exact slot-48 placement remains `heap-store-optimization -> redundant-set-elimination -> vacuum -> dae-optimizing -> inlining-optimizing`.

The closure commit is `676b90f68c2993899c1f5432a2d4253dead50774`. The current native release binary SHA-256 is `e6d703492f7008bd3265569e8620cfb6514cd1604e533e4bf497ed03bf8c066e`.

## Open release blocker

A new artifact-scale stress input was produced by applying Binaryen-v131 `-Oz` to the current `12,981,460`-byte wasm debug CLI. The resulting module is `3,597,499` bytes, has `5,374` defined functions, and contains roughly `251,000` rendered direct-call occurrences.

- The initial Starshine plain-DAE implementation exceeded the bounded `1500`-second run without producing output.
- Cached current call counts, dependency graphs, boundary facts, dropped-result prechecks, and committed work applications now complete the same run in `131.195s` with valid output SHA-256 `fa73c6aac3c93ac9c97bd261ce24403c4b7cfb8fbdb417f4ed74e1488caf11a9`.
- Binaryen v131 `--dae` completes the same input in `0.356s`.

The timeout is resolved and repeated clean Starshine runs remain byte-identical, but the remaining pass-local gap is still a DAE-owned dense-call performance blocker. It is not a semantic, validation, generated-parity, or size-classification failure. Track the remaining fixed-loop and selected-lane cost under `[O4Z-DAE-PERF]001` in `agent-todo.md`.
