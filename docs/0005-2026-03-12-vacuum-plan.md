# Vacuum Performance Plan

Goal: bound worst-case `Vacuum` time without weakening correctness.

## Current Diagnosis
- Full-function seeding + optimization + final checks happen before budget gating can stop work.
- Analysis caches are keyed by expression value (`Map[TInstr]`, `Map[TExpr]`) which can be expensive on deep trees.
- Multiple helper calls walk large subtrees repeatedly (`type`, effects, label rebasing, value-break checks).
- `top_instrs` is a weak cost signal for pathological nesting.

## Strategy

### Stage 0 — Instrumentation
- Add shape scan before seed:
  - total nodes
  - depth
  - block/if/drop counts
  - wrapper density
  - result-typed block count
- Add per-phase timer and top-N hotspot reporting.
- Distinguish normal/cheap/skip reasons in logs.

### Stage 1 — Better Guardrails
- Budget seed, optimize, and final check as one coherent budget.
- Replace `top_instrs` threshold with node-depth aware cost estimate.
- Add deterministic cheap mode for expensive shapes:
  - keep obvious drop/no-op simplifications
- Emit explicit `pathological_shape`, `seed_budget_exceeded`, and `mode=cheap_only`.

### Stage 2 — Replace Value-Keyed Caches
- Assign dense node ids in prepass.
- Store facts in arrays:
  - type/stack summary
  - effect summary
  - control-transfer / break-depth metadata
- Compute facts in a fused post-order pass instead of multiple independent recursive helpers.

### Stage 3 — Remove Repeat Subtree Walks
- Replace hot `has_break_to_depth...`, label-rebase transforms, and value-break LUB scans with cached metadata.
- Avoid generic type/effect helpers in common rewrite loops; prefer cached facts.

### Stage 4 — Tiered Vacuum
- Keep full `Vacuum` as primary path.
- Keep `cheap_only` fallback for high-cost classification.
- Preserve correctness via final validation and preserved invariants.

## Target End State
1. Deterministic, bounded first expensive `Vacuum` pass.
2. Clear reason when degradation engages.
3. Lower per-function complexity through indexed facts.
4. Fewer full-subtree rescans in the rewrite loop.

## Suggested Order
1. Instrument + shape classifier.
2. Budget + cheap path.
3. Value-keyed cache replacement.
4. Subtree-scan elimination.
5. Final benchmark gates + regressions.

## Risk Controls
- Keep existing fixtures for stack signatures and control-flow safety.
- Do not add more feature-specific heuristics before baseline metrics improve.
- Never relax rewrite legality checks while optimizing performance paths.
